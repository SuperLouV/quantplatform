# Project Context

最后更新：2026-04-27

## 项目定位

`QuantPlatform` 是一个以股票池驱动的量化研究与交易辅助平台。

当前不是实盘自动交易系统，当前阶段更接近：

- 数据与研究平台
- 股票池管理工具
- 策略研究与回测平台
- 人工下单前的分析与建议系统

后续再逐步扩展到：

- 模拟执行
- 半自动执行
- 与券商接口联动

## 账户背景

- `LongPort`：约 `20,000 USD`
- `IBKR`：约 `5,000 USD`

当前使用方式：

- 前期以人工下单为主
- 系统先负责研究、筛选、建议、风险与回测
- 当前不做真实自动下单

## 当前产品方向

当前产品主线是：

1. 股票池来源与管理
2. 数据接入与标准化
3. 个股快照与分析输出
4. 策略研究
5. 回测验证
6. 后续执行接口

前端产品形态参考券商终端，但当前只保留最小必要结构：

- 股票列表
- 主工作区
- 个股图表
- 右侧新闻 / AI / 风险区域

## 当前技术原则

- API 凭证只放本地环境，不写入仓库，不写入前端
- 前端只调用本地后端接口
- README、计划、日志要持续更新
- 分层必须清晰，便于后续替换数据源和接券商
- 先保证系统结构可扩展，再逐步增加策略复杂度

## 当前数据策略

当前已接入：

- `yfinance`
- FRED release calendar
- Fed FOMC 日历
- Census 经济指标日历

当前已规划但未正式接入：

- `IBKR`
- `SEC EDGAR / 13F`
- `FINRA`
- `Polygon.io`
- `Finnhub`
- `Twelve Data`
- `Alpha Vantage`
- 舆情监控源：X/Twitter、Reddit、StockTwits、新闻

数据层现状：

- 已有基础快照、历史数据、原始层与处理层
- 已有速率限制、重试、超时、失败保护和本地操作日志
- 已有美股交易日历和最近已完成交易日判断
- 还缺 provider 降级和更可靠的生产级行情源

## 当前策略方向

目前还没有正式定稿的交易策略。

当前已确认的是：

- 第一版先做简单策略，不追求复杂
- 更适合低频、日线、趋势/波段型研究
- 当前 `trend_momentum_v1` 是 Scanner 策略，只用于生成候选观察列表
- Trading 策略必须等风控和回测验证后再推进

当前 Scanner 策略骨架：

- 截面动量排名
- 趋势确认
- RSI 健康度
- 成交量参与
- 数据质量和事件风险降级

## 当前最重要的问题

当前核心优先级是：

1. 策略定义
2. 风控建议
3. 每日报告
4. 回测框架
5. provider fallback 和后续数据源
6. IBKR / LongPort 执行层预留

当前不要被这些事情分散：

- 过早做真实自动交易
- 过早追求复杂 UI 页面数量
- 过早引入太多数据源
- 过早做过于复杂的 AI 推理层
