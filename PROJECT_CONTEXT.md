# Project Context

最后更新：2026-04-24

## 项目定位

`QuantPlatform` 是一个以股票池驱动的量化研究与交易辅助平台。

当前不是实盘自动交易系统，当前阶段更接近：

- 数据与研究平台
- 股票池管理工具
- 策略研究与回测平台
- 人工下单前的分析与建议系统

后续再逐步扩展到：

- 模拟执行
- 半自动执行
- 与券商接口联动

## 账户背景

- `LongPort`：约 `20,000 USD`
- `IBKR`：约 `5,000 USD`

当前使用方式：

- 前期以人工下单为主
- 系统先负责研究、筛选、建议、风险与回测
- 当前不做真实自动下单

## 当前产品方向

当前产品主线是：

1. 股票池来源与管理
2. 数据接入与标准化
3. 个股快照与分析输出
4. 策略研究
5. 回测验证
6. 后续执行接口

前端产品形态参考券商终端，但当前只保留最小必要结构：

- 股票列表
- 主工作区
- 个股图表
- 右侧新闻 / AI / 风险区域

## 当前技术原则

- API 凭证只放本地环境，不写入仓库，不写入前端
- 前端只调用本地后端接口
- README、计划、日志要持续更新
- 分层必须清晰，便于后续替换数据源和接券商
- 先保证系统结构可扩展，再逐步增加策略复杂度

## 当前数据策略

当前已接入：

- `yfinance`

当前已规划但未正式接入：

- `IBKR`
- `FRED`
- `SEC`

数据层现状：

- 已有基础快照、历史数据、原始层与处理层
- 还缺更完整的数据工程能力：
  - 速率限制
  - 重试机制
  - 超时处理
  - 失败保护
  - provider 降级

## 当前策略方向

目前还没有正式定稿的交易策略。

当前已确认的是：

- 第一版先做简单策略，不追求复杂
- 更适合低频、日线、趋势/波段型研究
- 先构建：
  - 规则化买点
  - 规则化卖点
  - 风险等级
  - 回测闭环

当前最可能的第一版策略骨架：

- 趋势
- 动量
- 简单止损

## 当前最重要的问题

当前核心优先级是：

1. 策略定义
2. 数据层工程化
3. 技术指标与规则信号
4. 风控建议与每日报告
5. 回测框架
6. IBKR / LongPort 执行层预留

当前不要被这些事情分散：

- 过早做真实自动交易
- 过早追求复杂 UI 页面数量
- 过早引入太多数据源
- 过早做过于复杂的 AI 推理层
