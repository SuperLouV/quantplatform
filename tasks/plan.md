# Plan

日期：2026-04-24

## 项目目标

构建一个以股票池驱动的数据分析产品，底层具备研究、回测和后续半自动执行能力，优先支持免费或低成本数据接口。

## 路线原则

- 先把股票池、数据层、分析层的产品对象设计清楚
- 先接免费或低成本数据接口，再考虑券商执行接入
- 先做日线数据、股票池管理和分析输出，再进入回测和执行
- 第一阶段重点不是收益目标，而是形成稳定的股票池驱动分析闭环

## 当前阶段

当前处于 Phase 1 到 Phase 2 的过渡阶段，重点是补齐数据底座，同时把股票池对象和产品输出层正式建立起来。

## 分阶段路线图

### Phase 0：项目初始化

- 完善 `src/quant_platform/` 包结构
- 增加基础配置加载
- 增加统一数据模型
- 设计任务、日志和文档管理方式
- 明确模块边界：数据、指标、筛选、评分、回测、执行

本阶段交付物：

- 项目目录结构
- 基础配置模板
- 计划、路线图和工作日志
- 第一版模块设计

### Phase 1：数据与参考信息底座

- 定义 `provider` 抽象层
- 接入首个历史日线数据源
- 接入基础参考数据：股票列表、行业、市值、财报日期
- 统一 OHLCV、股票元数据、事件日历字段
- 增加重试、限频、缓存、错误处理
- 形成本地数据落地方式：`raw` / `processed` / `cache`

本阶段交付物：

- 可拉取并保存美股历史日线
- 可更新基础股票池参考数据
- 有统一数据模型，后续模块可直接复用
- 明确第一版数据源替换成本和扩展方式

### Phase 2：股票池管理与研究输出

- 定义 `StockPool` 文件结构和产品模型
- 支持手动池、程序生成池和后续 AI 候选池
- 定义 `StockSnapshot` 输出模型
- 建立选中股票池后的标准数据输出契约
- 支持生成 `theme_pool`、`system_pool`、`watchlist`、`tradable_universe`

本阶段交付物：

- 独立的股票池对象与文件结构
- 可供界面读取的股票池快照
- 单只股票的标准化摘要输出模型

### Phase 3：策略验证与回测

- 定义回测输入输出结构
- 实现信号驱动的回测运行器
- 增加交易成本、滑点和调仓逻辑
- 输出收益、回撤、胜率、换手率等指标
- 支持样本内和样本外验证

本阶段交付物：

- 能稳定运行的回测框架
- 至少一条可复现的研究策略链路
- 统一的回测结果输出

### Phase 4：分析台与 AI 分析层

- 增加本地网页分析台
- 展示股票池、个股快照和分析结果
- 定义 `AIAnalysis` 结构化输出
- 支持风险等级、建议动作和关键原因展示
- 为后续问答和解释型 agent 预留接口

本阶段交付物：

- 可浏览股票池和个股详情的分析界面
- 可展示的 AI 分析结果对象
- 面向界面和 agent 的统一输出层

### Phase 5：模拟执行与半自动执行

- 接入券商或模拟盘接口
- 生成买卖建议和执行计划
- 增加仓位控制、止损、熔断等风控
- 增加日志、告警和任务调度
- 支持人工确认后执行

本阶段交付物：

- 可运行的模拟交易流程
- 半自动执行闭环
- 运行记录和复盘基础设施

## 第一阶段重点

当前最重要的是完成 Phase 1，并把 Phase 2 的产品对象层落地。

优先顺序：

1. 历史日线行情接口
2. 股票基础信息与 reference 数据
3. 股票池对象模型和文件结构
4. `StockSnapshot` 输出模型
5. 增量更新和产品产物路径约定

## 当前待办

- 完成股票池对象模型和目录规范
- 完成产品输出层 `StockPool / StockSnapshot / AIAnalysis`
- 接入基础信息同步和批量股票池更新
- 定义面向界面和 agent 的标准数据契约

## 暂不急于确定

- 实盘资金规模
- 第一版主策略参数细节
- 券商接入方式
- 执行层细节

这些可以在数据与回测闭环具备后再细化。
